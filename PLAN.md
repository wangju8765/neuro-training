## 技术决策记录

### 已确认的架构决策

1. **项目类型体系（当前）**
   - 两个类型：跟练型（routine）/ 自助打卡（self）
   - 工具不再作为独立类型，而是通过 `toolLink` 绑定
   - 有 toolLink 的项目点击后先打开工具页，完成后自动标记+加金币
   - 设置页「🧰 工具中心」可独立使用工具，不加金币不打卡
   - 兼容旧数据：type==='tool' 运行时自动转为 self + toolLink

2. **节拍器设计原则**
   - 进入时加载项目预设 BPM 和时长
   - 使用中可通过输入框和预设按钮自由调整（不改项目配置）
   - 节奏反馈：Web Audio API 两拍一组（强拍 1000Hz/0.45, 弱拍 660Hz/0.25）
   - 视觉辅助：文字节奏提示（●强·○弱）

3. **数字盘设计（v3.1 更新）**
   - 8组练习（左脚4组 + 右脚4组），每组 3位→4位→5位→6位
   - 每级个数：[4,6,8,8]，单脚26个，双脚52个
   - **严格跨中线约束**：禁止同号连续 + 禁止同侧连续 + 中线桥接
   - **手动控制**：用户按按钮逐位推进，不再自动跳
   - **语音分离**：引导语用阿里云 TTS（sitong 童声），数字播报用浏览器 TTS

4. **项目库与权限（v4）**
   - 公共项目库：所有 items 跨用户共享，通过 `active` 字段控制可见性
   - 收藏系统：用户从项目库 ⭐ 挑选项目到首页，`user_favorites` 表持久化
   - 管理员(PIN=8765)：创建/编辑/删除/禁用项目、管理奖励
   - 普通用户：浏览项目库、收藏、打卡、使用工具
   - 详情页统一入口：所有项目点击进入详情页，防误触完成

5. **数据同步策略（v4 更新）**
   - localStorage 保留用于 DC.items/PS/设置等基线数据
   - 每日完成记录：`daily_logs` 逐条实时写 Supabase，不做本地聚合
   - 收藏：Supabase 为主，localStorage 兜底
   - 同步失败不阻塞 UI

6. **移动端设计**
   - rem + 视口缩放 `calc(100vw / 375 * 16)`
   - 明亮配色（#f5f7fa 背景）
   - 不设配速带，用户自由安排项目
   - 不内置神经功能评估
