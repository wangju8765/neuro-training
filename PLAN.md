## 技术决策记录

### 已确认的架构决策

1. **项目类型体系（当前）**
   - 两个类型：跟练型（routine）/ 自助打卡（self）
   - 工具不再作为独立类型，而是通过 `toolLink` 绑定
   - 有 toolLink 的项目点击后先打开工具页，完成后自动标记+加金币
   - 设置页「🧰 工具中心」可独立使用工具，不加金币不打卡
   - 兼容旧数据：type==='tool' 运行时自动转为 self + toolLink

2. **节拍器设计原则**
   - 进入时加载项目预设 BPM 和时长
   - 使用中可通过输入框和预设按钮自由调整（不改项目配置）
   - 节奏反馈：Web Audio API 两拍一组（强拍 1000Hz/0.45, 弱拍 660Hz/0.25）
   - 视觉辅助：文字节奏提示（●强·○弱）

3. **数字盘设计**
   - 8组练习（左脚4组 + 右脚4组）
   - 每组数字位数递增：3位→4位→5位→6位
   - 跨中线约束：右侧数字(1-4)不相邻，左侧数字(6-9)不相邻
   - 语音播报：逐位读数字 + 引导语，使用 Web Speech API

4. **移动端设计**
   - rem + 视口缩放 `calc(100vw / 375 * 16)`
   - 明亮配色（#f5f7fa 背景）
   - 不设配速带，用户自由安排项目到时段
   - 不内置神经功能评估

5. **数据同步策略**
   - localStorage 是基线（离线可用，秒级响应）
   - Supabase 是持久层（跨设备同步）
   - 写入：先写本地 → 异步同步到 Supabase
   - 读取：先读本地 → Supabase 数据覆盖非空字段
   - 同步失败不阻塞 UI
