## P0 — 第一批改动

### 改动1：项目类型改三选一

**现状：** 目前项目类型是「跟练型」「纯打卡」「🎵节拍器」三个平铺在一起，「纯打卡」和「节拍器」实际上是两个不同的东西——节拍器是工具型而不是打卡类型。

**改为：** 跟练型 / 工具型 / 自助打卡

**具体变化：**

| 旧类型 | 新类型 | 说明 |
|-------|-------|------|
| 跟练型（routine） | → 跟练型 | 不变 |
| 纯打卡（simple） | → **自助打卡**（self） | 功能扩展——加了步骤说明字段 |
| 节拍器（metronome） | → **工具型**下的一个子类 | 不再是独立类型，是工具型的一种 |

**项目编辑器 UI 变化：**

```
类型按钮： [跟练型] [工具型] [自助打卡]

选择「工具型」时显示：
  工具类型： [节拍器 ➡️ BPM=___] [数字盘] [其他（预留）]

选择「自助打卡」时显示：
  步骤描述：[多行文本框]  ← 新增
  道具清单：[可选，逗号分隔]  ← 新增
  (循环次数隐藏、节拍参数隐藏)
```

**代码改动点：**
- `swType(type)` — 加一种类型，隐藏/显示对应字段
- `saveItem()` — 自助打卡类型保存 `desc`（描述）和 `props`（道具列表）
- `openEditor()` / `fillEditor()` — 读写新字段
- 首页列表 `rMg()` — 显示新类型名称（符号+文字）

### 改动2：自助打卡详情页

**现状：** 纯打卡项目点击后弹出"已完成"确认。

**改为：** 自助打卡项目点击后显示项目详情页（步骤说明 + 道具清单 + 完成按钮）。

**页面结构：**
```
┌─────────────────────────┐
│ ← 返回                    │
│                          │
│ 🧠 Pencil Push-ups        │
│ ─────────────────────     │
│ 📦 道具：铅笔一支           │
│ ─────────────────────     │
│ 📝 步骤说明：               │
│ 1. 坐直，铅笔握在一臂距离    │
│ 2. 聚焦铅笔上的字母…        │
│ 3. …                      │
│                          │
│ [ ✅ 完成练习 ]             │
└─────────────────────────┘
```

**如果已经有步骤描述（desc字段），展示详情页；如果 desc 为空，弹出确认（兼容旧数据）。**

### 改动3：节拍器工具升级

**现状：** 节拍器固定 54BPM，手动修改。

**改为：**
- 项目编辑器创建/编辑工具型-节拍器时，可设置 BPM
- 节拍器页面标题显示当前项目名和 BPM
- 预置几个常用 BPM 快速切换按钮（40、50、54、60、72）

**预设方案：**
```
BPM快速选择：
[40 - 摇篮] [50 - 摇摆] [54 - 标准] [60 - 拍手] [72 - 进阶]
```

## P1 — 第二批改动

### 改动4：分类标签系统

**标签列表（7个）：**

| 标签 | Emoji | 颜色 | 涵盖项目 |
|------|-------|------|---------|
| 原始反射整合 | 🧩 | 紫色 | 海星呼吸、交叉爬行、雪天使、摇篮式摇摆等 |
| 前庭-小脑训练 | 🌀 | 青色 | 旋转脱敏、单脚站、摇摇摆、蹦床、球上平衡等 |
| 视觉训练 | 👁️ | 蓝色 | Pencil push-ups、Brock string、Saccade、VOR等 |
| 自主神经调节 | 🌬️ | 橙色 | 蜜蜂呼吸、冷水唤醒、4-7-8呼吸、ATM等 |
| 精细运动 | ✋ | 粉色 | 指尖对碰、橡皮筋拉伸、数字盘 |
| 节奏与时序 | ⚡ | 黄色 | 节拍器踏步、拍手同步、交叉触膝 |
| 姿势与本体感觉 | 🧘 | 绿色 | ATM训练、高低位交替、平衡板 |

**实现方式（轻量）：**

1. **数据结构层：** 项目对象加 `tag: string[]` 字段（1-2个标签）
2. **编辑器层：** 新建/编辑项目时显示标签选择器（多选）
3. **显示层：** 首页项目列表显示标签小徽章
4. **统计层：** 首页（或设置页）新增「本周统计」区块，显示各标签完成次数

### 改动5：已有项目批量迁移

把补充调研里的 49 个项目一次性导入为 APP 中的项目数据（带标签、带类型、带步骤描述），不依赖 Zeno 逐个手动创建。

**形式：** 一个 JS 片段，在开发者控制台跑一次，或者在应用启动时检测到首次运行则插入。更推荐：直接在 `DC.items` 默认初始化数据里预置。

## 三、数据结构的增量

### 项目对象（新增字段）

```javascript
// 现有字段不变，新增以下：

// 所有类型通用
item.tag = ['🧩原始反射']   // 标签数组，1-2个

// 自助打卡类型特有
item.type = 'self'           // 目前有 routine/simple/metronome
item.desc = "1. 坐直…\n2. …"  // 步骤说明（多行文本）
item.props = "铅笔,贴纸"      // 道具清单（逗号分隔）

// 工具型特有
item.type = 'tool'
item.toolType = 'metronome'  // 或 'numberpad'
// 节拍器时用现有 item.bpm, item.dur
```

## 五、已确认事项

### 步骤说明格式
**纯文本 + 换行分段**。用户在一个 `<textarea>` 中分行输入，显示时自动保留换行。不需要 Markdown。

### 统计页入口
两个入口：
- **入口A：** 设置页（`pS`）新增「📊 训练统计」行
- **入口B：** 首页顶部进度条区域可点击，点击跳转到统计页

### 标签规则
- **已有项目：** 我一性次打上标签，用户不需要自己编辑
- **用户新建项目：** 标签可选，不选则默认为空
- **统计时的兜底：** 无标签项目归入「📋 自建项目」类别，作为第8个统计条目
