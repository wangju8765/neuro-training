# 积分系统设计方案

## 1. 数据结构设计

### 1.1 积分总览（在 DC 对象中新增字段）

```json
{
  // ... 现有字段：slots, items, settings, trash ...

  "points": 0,                    // 当前总积分（整数）
  "rewards": [                     // 奖励项目配置
    {
      "id": "r_1718000000000",
      "name": "看20分钟动画片",
      "icon": "📺",
      "cost": 10,
      "createdAt": 1718000000000
    }
  ],
  "pointLog": [                    // 积分流水（按时间倒序）
    {
      "type": "earn",              // "earn" 或 "redeem"
      "amount": 2,
      "itemId": "crawl",
      "itemName": "交叉爬行",
      "balance": 42,               // 操作后的余额
      "timestamp": 1718000000000
    }
  ],
  "dailyPoints": {}                // 按日统计: { "2026-5-27": { "earned": 8, "redeemed": 10 } }
}
```

### 1.2 localStorage Key 规划（改动最小化）

| Key | 用途 | 变更 |
|-----|------|------|
| `dc_items` | 项目列表 | 不变 |
| `dc_slots` | 时段安排 | 不变 |
| `dc_settings` | 设置 | 不变 |
| `dc_trash` | 回收站 | 不变 |
| `dc_YYYY-M-D` | 每日完成 | 不变 |
| **`dc_points_state`** | **积分数据** | **新增** |

将积分相关数据单独存为 `dc_points_state`，不与 `DC` 对象耦合，避免和现有代码的 `flushDC()` 互相影响。

### 1.3 积分状态管理对象

```javascript
let PS = {
  points: 0,
  rewards: [],
  pointLog: [],
  dailyPoints: {}
};
```

存取函数：
- `initPS()` — 从 localStorage 读取 `dc_points_state`，初始化 PS
- `flushPS()` — 将 PS 写入 localStorage `dc_points_state`
- `getTodayPoints()` — 返回今日获得积分
- `getDailyKey()` — 返回 `"2026-5-27"` 格式的日期键

## 3. 需要修改的现有函数清单

| 函数 | 修改内容 |
|------|----------|
| `initDC()` | 无变化（积分独立存储） |
| 新增 `initPS()` | 从 localStorage 读取 `dc_points_state` |
| 新增 `flushPS()` | 写入 `dc_points_state` |
| 新增 `addPoints(amount, itemId, itemName)` | 加积分、记流水、更新 dailyPoints、flush |
| 新增 `spendPoints(amount, rewardName)` | 扣积分、记流水、flush |
| 新增 `getTodayPoints()` | 返回今日累计获得积分 |
| `rH()` (首页渲染) | 在进度卡片中新增一行显示今日积分和总积分 |
| `updateHeader()` | **不修改**（保持现有时段问候） |
| `upPr()` (进度更新) | 在进度卡片下方增加积分显示 |
| `sX()` (进入练习 - 纯打卡) | 打卡完成时调用 `addPoints(1, ...)` |
| `fEx()` (完成 - 跟练型) | 完成时调用 `addPoints(2, ...)` |
| `openSet()` (设置页) | 无需修改（只是显示设置页内容） |
| `rH()` + 设置页 render | 在设置页新增「积分商店」区块（两个入口：兑换奖励、管理奖励） |
| `goH()` | 确保返回首页时刷新积分显示 |

**不需要修改的现有函数：**
- 所有 editor 相关函数（`openEditor`, `saveItem`, `addStep` 等）
- `delItem`, `restoreItem`（项目删除/恢复不涉及积分）
- 时段安排相关函数（`openSl`, `rSl`, `openSE`, `svSE` 等）
- `openNE`, `saveNE`（姓名编辑）

## 4. 关键交互流程

### 4.1 纯打卡完成 → 积分

```
用户点击纯打卡项目
    ↓
sX() 发现 type==='simple'
    ↓
Dn.push(id); svDn();  ← 原有逻辑
addPoints(1, itemId, itemName);  ← 新增：+1 分
    ↓
显示完成弹窗（弹窗中显示 "+1 积分" 和今日累计）
```

### 4.2 跟练型完成 → 积分

```
fEx() 被调用（自动完成或用户点"我做完了"）
    ↓
Dn.push(id); svDn();  ← 原有逻辑
addPoints(2, itemId, itemName);  ← 新增：+2 分
    ↓
显示完成弹窗（弹窗中显示 "+2 积分" 和今日累计）
```

### 4.3 兑换奖励

```
用户进入积分商店 (pPt) → 查看奖励列表
    ↓
点击"兑换" → 弹出确认弹窗
    ↓
确认 → spendPoints(reward.cost, reward.name)
    ↓
积分扣减 → 记流水 → 刷新页面
    ↓
显示成功反馈
```

### 4.4 家长管理奖励

```
用户进入设置页 → 点击"管理奖励" → 进入 pRm 页面
    ↓
查看/编辑/删除现有奖励项
新增加载行表单 → 填写图标、名称、所需积分 → 保存
    ↓
数据写入 PS.rewards → flushPS()
    ↓
刷新列表
```

### 4.5 积分数据持久化

```
所有积分操作 → 修改 PS 对象 → flushPS() → localStorage.setItem('dc_points_state', JSON.stringify(PS))
                                        ↓
                                ps_init() → 启动时加载
```

## 6. 风险与注意事项

1. **避免与现有 DC 对象耦合**：积分数据单独存为 `dc_points_state`，不放在 `DC` 对象里，防止 `flushDC()` 覆盖积分数据
2. **启动顺序**：`initDC()` 和 `initPS()` 都需要在启动时调用
3. **每日统计**：`dailyPoints` 用于在首页展示「今日获得积分」，不与总积分混淆
4. **一致性**：`flushPS()` 每次积分变更后立即调用，不延迟
5. **UI 一致性**：所有新页面沿用现有明亮圆角卡片风格，与现有设计语言统一
